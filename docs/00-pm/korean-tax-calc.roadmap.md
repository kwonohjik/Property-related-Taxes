## Phase 1 — v1.0 MVP (3~4주)

**목표**: 양도소득세 계산기를 중심으로 핵심 인프라 구축

### 마일스톤

| Week | Milestone | 산출물 |
|------|-----------|--------|
| W1 | 프로젝트 기반 구축 | Next.js 15 프로젝트 + Supabase 연결 + DB 스키마 + `transfer` 세율 시딩 + rate limiting + zustand StepWizard 상태 관리 + vitest/Playwright 설정 |
| W1~W2 | 인증 시스템 | 이메일/구글/카카오 로그인 + 미들웨어 + 프로필 |
| W2~W3 | 양도소득세 계산 엔진 | 비과세 판단 + 일시적 2주택 특례 + 세액 계산 + 장기보유공제(중과시 배제) + 중과세(미등기 70% 포함) + 감면 + 환산(공식 기반) + 보유기간 정밀 계산 |
| W3~W4 | 양도소득세 UI + 이력 | 다단계 입력 폼 + 결과 표시 + 이력 저장/조회 |

### 핵심 기능

- 비로그인 사용자 계산 허용 (이력 저장·PDF만 로그인 필요, 비로그인 계산 결과는 sessionStorage 임시 보관 → 로그인 시 자동 저장)
- 1세대1주택 비과세 판단 (12억 기준) + **일시적 2주택 비과세 특례**
- 누진세율 6~45% 자동 계산
- 장기보유특별공제 (최대 80%, **중과세 대상 시 배제**)
- **미등기 양도 70% 단일세율** + 비사업용토지·다주택 중과세
- 조세특례제한법 감면 4종 (상세 조건 포함)
- **조정대상지역 판단 시점 구분** (비과세=취득일, 중과=양도일)
- **보유기간 정밀 계산** (취득일 다음날 기산, 윤년 처리)
- 기준시가 **수동 입력** + 부동산공시가격알리미 외부 링크 안내 (자동 조회는 v1.4)
- 취득가액 환산 (수동 입력된 기준시가 기반)
- Supabase Auth 이메일/소셜 로그인
- 계산 이력 저장/조회 (RLS 적용)
- StepWizard 상태 중간 저장 (sessionStorage)
- Vercel Edge Middleware rate limiting (계산 API: 분당 30회)

### 기술 결정 사항

| 항목 | 결정 | 근거 |
|------|------|------|
| Framework | Next.js 15 (App Router, Turbopack) | React 19 지원, 향상된 캐싱 모델, 2026 Q3 안정 버전 |
| 세율 시딩 범위 | `transfer`만 (나머지는 해당 Phase에서) | 미사용 데이터 선투입 방지, 세법 변동 대응 |
| 기준시가 | 수동 입력 (v1.4에서 API 자동 조회 전환) | 기준시가 API 연동 복잡성을 MVP에서 분리 |
| 폼 관리 | react-hook-form + zod | 6가지 세금 × 다단계 폼의 복잡한 유효성 검증 |
| 상태 관리 | zustand + sessionStorage persist | 새로고침/이탈 시 입력 데이터 유실 방지, StepWizard 전역 상태 |
| 날짜 계산 | date-fns | 보유기간·거주기간·세율 시점 조회 정밀 계산 |
| Rate limiting | Upstash Redis + @upstash/ratelimit | 비인증 사용자 API 남용 방지 |
| 비로그인 정책 | 계산은 비로그인 허용, 이력 저장·PDF만 로그인 필요 | 전환율 극대화 + 보호 라우트 분리 |
| 모니터링 | @sentry/nextjs | 계산 엔진 런타임 오류 감지 |
| DB 마이그레이션 | Supabase CLI migration | 스키마 변경 이력 관리, 멱등성 보장 |
| Type-safe DB | Supabase CLI (`supabase gen types`) | DB 스키마 → TypeScript 타입 자동 생성 |
| 테스트 | vitest + @testing-library/react + Playwright | 계산 정확도 99%+ 검증, 다단계 폼 E2E 테스트 |
| Language | TypeScript 5.x (strict mode) | 세금 계산 정확도를 위한 타입 안전성 |
| Runtime | Node.js 22 LTS | 2026 시점 Active LTS |
| 정밀 연산 | 원(정수) 단위 연산 + 세금별 절사 규정 준수 (과세표준: 천원/만원 미만 절사, 세액: 원 미만 절사). 비율 연산 시 곱셈 먼저 후 나눗셈. 고가 물건 환산 시 중간값 오버플로우 방지 (BigInt 또는 분할 계산) | JS 부동소수점 오류 방지 + 국세청 예시와 1원 단위 일치 |
| Supabase keepalive | Vercel Cron Job (6일 주기) | Free tier 1주 미사용 자동 pause 방지 |
| 이력 보존 | 사용자당 최대 200건 | Supabase Free 500MB 내 안정 운영 |
| 세율 관리 | 시딩 CLI 스크립트 (`npm run seed:tax-rates`) | 세법 개정 시 배포 없이 DB 업데이트, v2.0에서 Admin UI |
| 에러 처리 | App Router error.tsx + 계산 엔진 에러 코드 체계 | 사용자에게 구체적 오류 안내, 입력 데이터 유실 방지 |

### 완료 기준

## Phase 3 — v1.2 (3~4주)

**목표**: 상속세·증여세 계산기 + 재산 평가 기능

### 마일스톤

| Week | Milestone | 산출물 |
|------|-----------|--------|
| W0 | `inheritance`, `gift` 세율 시딩 | 상속세·증여세 누진세율 + 공제한도 + 인적공제 + 관계별 증여공제 데이터 |
| W1~W2 | 상속세 계산 엔진 | 누진세율 + **사전증여 합산** + 상속공제 7종(인적공제 4종 포함) + **공제 종합한도** + 배우자공제(실제상속분/법정상속분) + 법정상속분(잔여분 처리) + 세대생략 할증(안분) + **세액공제 순서** |

### 핵심 기능

- 상속세 누진세율 (10~50%) + 공제 최적화 (일괄공제 vs 항목별 공제 자동 비교, **인적공제 4종 상세 산출**)
- **사전증여재산 합산** (상속인 10년 내, 비상속인 5년 내) + 기납부 증여세 공제
- 배우자상속공제 (`min(max(실제상속분, 5억), min(법정상속분, 30억))`, 상속인 구성 입력 기반)
- **상속공제 종합한도** (과세가액 초과 불가)
- 증여재산공제 (관계별 차등, 10년 합산 **잔여 한도 계산**)
- **증여세 비과세 판단** (생활비·교육비 등 사회통념상 비과세)
- 재산 평가 (시가·보충적 평가 안내)
- 세대생략 할증 (30%/40%, **안분 계산**)
- **세액공제 적용 순서** (할증 → 기납부 → 신고공제 3%)
- 세액 1,000만원 초과 시 **분납·물납 가능** 안내
- 미성년자 증여 시 **증여자 연대납세의무** 안내

### 완료 기준

## Phase 4 — v1.3 (3~4주) — 핵심 연동 Phase

**목표**: 재산세 + 종합부동산세 구현 + **재산세↔종부세 자동 연동**

### 마일스톤

| Week | Milestone | 산출물 |
|------|-----------|--------|
| W0 | `property`, `comprehensive_property` 세율 시딩 | 재산세·종부세 세율 + 공정시장가액비율 + 공제율 |
| W1 | 재산세 계산 엔진 | 주택·토지·건축물 세율 + 1세대1주택 특례 + 세부담 상한 + 부가세 |
| W1~W2 | 재산세 UI | 물건 유형별 입력 폼 + 전년도 세액 선택적 입력 + 결과 표시 |
| W2~W3 | 종합부동산세 계산 엔진 | 공시가격 합산 + 공제 + 누진세율 + **재산세 공제 연동** + 세부담 상한 |
| W3 | 종합부동산세 UI | 다주택 목록 입력 (중간 저장) + 합산 실시간 프리뷰 + 연동 결과 표시 |
| W3~W4 | 연동 통합 테스트 | 재산세↔종부세 공제 연동 검증 + 엣지케이스 |

### 핵심 기능

**재산세:**

- 주택 재산세 누진세율 (0.1~0.4%)
- 1세대1주택 특례세율 (0.05%p 인하)
- 토지 재산세 (종합합산/별도합산/분리과세)
- 세부담 상한 (전년 대비 105~130%)
- 부가세 합산 (지방교육세, 지역자원시설세, 도시지역분)

**종합부동산세:**

- 인별 공시가격 합산 과세
- 기본공제 (9억, 1세대1주택 12억)
- 공정시장가액비율 적용 (DB에서 로드, 현행 60%)
- 누진세율 (0.5~2.7%)
- 1세대1주택 고령자 공제 (20~40%) + 장기보유 공제 (20~50%) = 최대 80%
- **재산세 비율 안분 공제** (핵심): `공제할 재산세 = 재산세 부과세액 × (종부세 과세표준 ÷ 재산세 과세표준)` — 단순 전액 차감이 아닌 비율 안분 (종부세법 시행령 제4조의2)
- 세부담 상한 (150%/300%)
- 농어촌특별세 (종부세의 20%)
- 토지분 종합부동산세

### 세부담 상한 — 전년도 세액 처리 전략

| 케이스 | 처리 방식 |
|--------|----------|
| 전년도 세액 미입력 | 세부담 상한 계산 **생략**, 산출세액 그대로 표시 + "전년도 고지서 세액 입력 시 상한 적용 가능" 안내 |
| 전년도 세액 직접 입력 | 사용자가 전년도 고지서 참조하여 입력 → 상한 계산 적용 |
| 앱 내 이전 이력 존재 | 이전 계산 결과에서 자동 채움 제안 (확인 후 적용) |

## Dependencies & Risks

| 구분 | 내용 | 대응 |
|------|------|------|
| **외부 의존** | Supabase (Auth+DB), Vercel (배포), Upstash Redis (캐싱+rate limit), 국토부 API (기준시가) | 모두 무료 플랜으로 MVP 시작 |
| **핵심 리스크** | 재산세↔종부세 연동 복잡성 | Phase 4를 3~4주로 배정 + 모드 A만 우선 구현 + 통합 테스트 우선 |
| **세법 변동** | 공정시장가액비율, 중과세 유예 여부 | DB 기반 관리로 배포 없이 대응 |
| **PDF 생성 제약** | Vercel Serverless 번들·시간·메모리 제한 | Phase 2 첫 주에 기술 PoC로 검증 후 방식 확정 |
| **API 남용** | 비인증 사용자 대량 호출 | Upstash rate limiting (계산 30/min, 기준시가 10/min) |
| **DB 용량** | 기준시가 전국 데이터 Supabase 500MB 초과 | API 프록시 방식 전환 + 파일 적재 범위 수도권+광역시 한정 |
| **세부담 상한 데이터** | 전년도 세액 부재 | 선택적 입력 + 미입력 시 상한 생략 안내 |
| **조정대상지역 변경** | 정부 수시 지정/해제 | `regulated_areas` 테이블 DB 관리, 배포 없이 업데이트 |
| **세무사 검증** | 검증 케이스 확보 | 국세청 예시 + 세무사 실무사례집 기반, 외부 권위 소스 활용 |
| **과거 세율 조회** | 과거 시점 계산 지원 | effective_date 기준 시점 조회, 과거 세율 보존 |
| **부동소수점 오류** | JS 부동소수점 한계로 세금 계산 오차 | 원(정수) 단위 연산 원칙 + 세금별 절사 시점 준수 (과세표준 천원/만원 절사, 세액 원 미만 절사) |
| **Supabase pause** | Free tier 1주 미사용 시 자동 pause | Vercel Cron Job 6일 주기 keepalive ping |
| **이력 DB 증가** | calculations 테이블 무한 증가 | 사용자당 200건 상한 + 자동 정리 |
| **국토부 API 쿼터** | 공공데이터포털 일일 1,000회 한도 | Redis 24h 캐싱 + 쿼터 초과 시 수동 입력 fallback |
| **세율 관리 도구** | 세법 개정 시 DB 업데이트 수단 필요 | v1.0: 시딩 CLI 스크립트, v2.0: Admin UI |
